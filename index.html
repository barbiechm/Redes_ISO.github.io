Tipos de Redes

¿Qué es una Red Informática?

Una red informática es un conjunto de dispositivos interconectados que permiten compartir recursos e intercambiar información entre sí. Esta interconexión puede realizarse de diversas maneras, ya sea a través de cables (como en una red LAN) o de forma inalámbrica (como en una red Wi-Fi). Las redes pueden ser locales (LAN), de área metropolitana (MAN) o de área amplia (WAN), e incluso globales como Internet.

Importancia de las Redes en Informática

  • Comunicación eficiente: Las redes permiten que los usuarios y sistemas se comuniquen de manera rápida y eficiente.
  • Acceso compartido a recursos: Permiten que múltiples usuarios compartan recursos como impresoras, archivos y aplicaciones.
  • Escalabilidad y flexibilidad: Las redes permiten la expansión sencilla de recursos y servicios.
  • Seguridad: A través de las redes, se implementan medidas de seguridad como cortafuegos y encriptación de datos.
  • Eficiencia y ahorro: Permiten la centralización de servicios y recursos, mejorando la eficiencia y reduciendo costos.
  • Conectividad global: Gracias a las redes, las computadoras se conectan globalmente, facilitando el comercio electrónico y las redes sociales.

Capas del Modelo ISO/OSI

  1. Capa 1: Física +

    Responsable de la transmisión de bits a través de un medio físico, como cables de cobre, fibra óptica o señales inalámbricas.

    Ejemplos: Ethernet, USB, Bluetooth.

  2. Capa 2: Enlace de Datos +

    Gestiona la transferencia confiable de datos entre nodos conectados directamente. Incluye detección y corrección de errores.

    Ejemplos: MAC (Control de Acceso al Medio), switches.

  3. Capa 3: Red +

    Encargada del enrutamiento de paquetes entre redes diferentes. Define direcciones lógicas (IP) y determina la mejor ruta.

    Ejemplos: IP, routers.

  4. Capa 4: Transporte +

    Gestiona la comunicación confiable entre dispositivos, incluyendo control de flujo y corrección de errores.

    Ejemplos: TCP (Transmission Control Protocol), UDP (User Datagram Protocol).

  5. Capa 5: Sesión +

    Establece, mantiene y termina sesiones entre aplicaciones. Gestiona la sincronización y el diálogo.

    Ejemplos: NetBIOS, RPC (Llamada a Procedimiento Remoto).

  6. Capa 6: Presentación +

    Traduce, cifra y comprime los datos para que sean compatibles entre sistemas.

    Ejemplos: SSL/TLS (cifrado), JPEG (compresión).

  7. Capa 7: Aplicación +

    Proporciona una interfaz para que las aplicaciones accedan a los servicios de red.

    Ejemplos: HTTP, FTP, SMTP.
